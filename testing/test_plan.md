#  План тестирования проекта NutriStore Manager (MVP)

## 1\. Цели тестирования

Основная цель тестирования - обеспечить соответствие разработанной бизнес-логики (Java/Spring Boot) функциональным требованиям и убедиться в стабильности взаимодействия с базой данных PostgreSQL/MySQL.

| **Тип тестирования** | **Что проверяется (Цель)** |
| :-- | :-- |
| **Модульное тестирование** | Корректность работы изолированных методов сервисов (/service), особенно расчёт цен, скидок, обработка сроков годности и проверка остатков склада. |
| **Интеграционное тестирование** | Взаимодействие между REST-контроллерами (/controller) и репозиториями (/repository), а также успешное выполнение CRUD-операций с БД. |
| **Функциональное тестирование (API)** | Сквозные бизнес-процессы: полный цикл оформления заказа, создание товара с корректными параметрами, работа модуля /analytics. |
| **Тестирование безопасности** | Проверка входных данных на уязвимость (санитизация), а также контроль доступа к критическим эндпоинтам (например, удаление данных, доступ к отчетам). |


## 2\. Среда и инструменты тестирования

| **ОС:** | Любая, поддерживающая Java (Windows/Linux/macOS). |
| :-- | :-- |
| **Стек:** | Java 17, Spring Boot 3, Maven. |
| **База данных:** | PostgreSQL 15+ (тестовый инстанс). |
| **Инструменты:** | JUnit 5, Mockito, Postman, Swagger UI. |

## 3\. Критические функции для тестирования

- **Управление товарами (CRUD):** Проверка создания товара с нулевым остатком и последующей поставки.
- **Обработка заказа:** Расчёт итоговой суммы с учетом промокода и автоматическое уменьшение остатка на складе.
- **Аналитика:** Генерация отчета по динамике продаж за указанный период.
